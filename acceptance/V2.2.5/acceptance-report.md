- 入口：同 §2 添加工时弹窗（多任务选项任务：「项目会议-项目启动会、项目内审会 | 1H」#57378）
- 复现步骤：
  1) 打开任务 #57378（含两个子项：项目启动会、项目内审会）的添加工时弹窗
  2) 子项「项目启动会」填 1h + 描述「V2.2.5验收⑥：启动会正常填写」；子项「项目内审会」填 **0h 且描述留空**
  3) 点「立即创建」
- 预期：提交成功；花费工时为 0 的项系统自动填写「暂无」
- 实际：提交成功（0 工时子项**免必填描述已生效** ✅，未再弹"请填写…完成情况"拦截）；但回读落库记录为「项目内审会[0h]：」——**描述为空，没有自动补"暂无"** ❌
- 交叉验证：前端 `submitForm` 源码中校验逻辑为"仅当子项工时>0 时才要求描述"（已实现），但组装 daily_work 时对 0 工时空描述项直接拼空串，未见补"暂无"逻辑；接口请求体与落库均为空串
- 严重度：一般（功能主体可用，发版描述中"自动填写暂无"这半句未实现；不影响提交）
- 附带正向验证：0 工时项描述**有填**时校验也不拦（同弹窗另一场景「项目内审会[0h]：」空 与 有值均可提交）；工时>0 描述留空仍被拦截「请填写…（花费工时不为 0 时必填）」✅ 校验口径正确
- 截图：final-06-花费0自动填暂无.jpg（可见 [0h] 项描述为空的落库现场）

### 版本详情块

```
V2.2.5  发布于2026-07-10
影响强度
强度中等，新增项目递交与自制发包任务管理，优化工时填写、组群创建与任务描述展示体验。

新增功能
需关注人员：PM、TB\BD、QA
1.「项目-项目递交」：（减少跨模块翻查递交历史的时间）
项目路由下新增项目递交模块，按时间轴汇总该项目全部递交记录，
支持按时/超时/延期/待递交状态统计与最新最早排序，
方便PM与商务在项目内直接跟踪递交进展；

需关注人员：DTA、PM
2.「模型外包-自制发包」：（统一沉淀自制建模任务数据，量化资产制作质量）
DTA组长创建内部建模任务时，仅需将任务关联到对应发包，
相关人员可在项目-模型外包中对"自制"类型发包下的任务进行系统化管理，
发包申请人可对每一个资产制作任务进行打分和评语；

体验优化
需关注人员：全员
1.「任务选项-项目内审会」
任务选项「项目管理-项目会议」下新增"项目内审会"，支持项目内审会议工时的规范登记；

2.「工时填写-截图粘贴上传」
填写工时上传截图支持剪贴板 Ctrl+V 直接粘贴，截图后无需保存本地文件再选择上传，减少填报操作步骤；

3.「创建任务-快速创建组群」
创建多人任务添加指派人时，支持就地新建组群并保存成员，顺手沉淀常用人员组合，无需事先到组群配置中维护；

4.「我的任务-日历填写花费」
点击日历中对应日期的任务项，直接弹出花费填写界面，右侧任务栏照常保留，更符合填报操作习惯；

5.「工时填写-零工时免填描述」
添加花费时，当日花费工时为 0 的项不再强制填写描述，仅工时大于 0 的项需填写当日完成情况；

6.「任务描述直接展示」
任务列表的任务描述直接展示描述内容，无需点击弹窗查看；
```
