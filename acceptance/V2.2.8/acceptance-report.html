只放已确认、无需产品拍板就必修的；本表只写产品人员能看懂的话，技术细节见 fix-handoff.md。

# 缺陷 严重度 建议
B1 模型数据看板「资产维度」明细接口在传入非法的负数分页大小时会返回服务器 500 错误（而非友好报错），同版发包维度明细同参却能优雅校验 轻微 后端对资产明细分页参数补负数/非法值校验，与发包明细口径对齐（返 code 而非 500）

修复交接：缺陷的复现步骤、定位线索与修复通过标准见 fix-handoff.md（给前后端开发，可整份投喂 AI）。

三、风险与待确认

# 事项 等级 建议
R1 递交超时筛选 is_over_tb_time 传非法枚举（如 abc）后端不校验，返回介于全量与超时集之间的部分集 中 后端补参数校验（非法值报错或按默认全量处理）
R2 「仅查看超时递交」筛选结果含实际早于要求递交时间的记录（如 #14076、#14057），是否应计入超时口径待产品确认 中 产品确认「超时」判定口径，必要时后端校准 is_over_tb_time 计算
R3 模型数据看板资产状态四档合计与资产总数存在差额（历轮实测 278 vs 316 差 38），疑漏「未开工」档统计 中 核对资产状态分档是否完整覆盖全部资产
R4 工作台 UGA 入口把登录态 token 明文拼接在跳转 URL query 上，存在被日志/Referer 泄露风险 中高 评估改用一次性票据 / POST 传递 / 短时效 token

未覆盖盲区声明：接口极限值（超大 limit、超长 keyword、特殊字符）未逐一穷举；需求自动暂停为时间驱动的后端定时任务，仅验证展示层与已落库样本，未从零构造「刚好满两周」的临界样本；UGA 跳转仅验证 window.open 携参，未登录 UGA 目标系统内页。

四、发版内容（初稿，待人工定稿）

V2.2.8 发布于 2026-07-23

影响强度

强度中等，新增项目人员看板、模型数据看板、需求「暂停中」状态与工作台 UGA 入口，并完善递交超时筛选与递交 API 版本选项。

新增功能

需关注人员：PM、PMO、DTA、全员

1.「统计-项目人员看板」：（减少人工汇总项目进展成本，提高人员项目盘点效率） 支持按项目管理、项目场景、项目开发、项目设计、项目技术、DTA资产等多维度查看成员项目分布与状态；项目管理维度纳入 QA 测试人员；项目场景维度点击场景名可下钻展开组员看板，逐人查看 7 天活跃、超周无进展、制作中、已营收、未开工等指标；

2.「统计-模型数据看板」：（沉淀模型发包数据，量化供应商资产交付质量） 支持模型数据总览与模型明细两个视角；总览汇总发包数量、发包金额、资产数量、资产评分、不合格资产占比与发包平均反馈数，并提供供应商质量评分排行榜与省份数量热力分布；模型明细支持按项目维度、资产维度下钻查看；

3.「项目-项目需求」新增「暂停中」状态：（自动识别停滞需求，减少人工盘点需求进展） 项目下进行中的新增、变更、初始需求，若任务最近结束日期距当前系统日期超过两周、且期间无新增任务、无工时填报，需求状态自动转为「暂停中」；

4.「我的工作台-UGA」：（打通 UGA 快捷访问） 开放 UGA 入口，支持携登录态跳转 UGA 正式、hzh、wsm 多环境；

体验优化

需关注人员：QA、PM

1.「递交列表」新增「仅查看超时递交」筛选，与递交状态下拉解耦，方便 QA 快速筛出超时递交记录； 2.「递交审批」WDPAPI 引擎版本选项补充 API2.3.x，方便 PM 审批递交申请时选择对应引擎版本；
