# 51PM — 操作库 / Playbooks（功能标准操作流程 SOP）

> **这是什么**：51PM 每个功能「怎么从头做完一件事」的标准操作序列（SOP）。
> **与 [entry_map.md](../entry_map.md) 的分工**：entry_map 回答「入口在哪」（一行索引 + 坑）；本目录回答「整件事怎么做完」（可复现的分步流程 + 直调接口 + 读数锚点）。找位置查 entry_map，要复现操作看这里。
>
> **为什么要有**：验收时 AI 已经把每个功能的真实流程走通了，走完就丢在一次性报告里太可惜。把它提炼成常驻资产后：
> - 未来「让 AI 定时查工时统计 / 数据统计」等任务，浏览器进去照着 playbook 就能复用，不用重新摸索；
> - 人（新同事、PM）也能拿来当操作手册；
> - 是可以外发、跨团队复用的产品资产。

## 谁来维护

由验收流程 [SKILL.md](../SKILL.md) **阶段 4（沉淀）** 自动回填：每轮验收把当轮真实走通的功能流程提炼成/更新为一个条目。也可手工补充历史功能。

## 目录约定

- 一个功能 = 一个 `.md` 文件，文件名用功能中文名（如 `填写工时.md`）。
- 同一业务域的多个小动作可合并进一个文件，用二级标题分节。
- **写操作类**在标题标 `【写】`，只读类标 `【只读】`，方便定时/批量任务筛选。

## 通用前置（每个 playbook 默认继承，不再各自重复）

1. **进站复用页签**：先找已开的 51pm 页签复用、页内导航到目标路由，未命中才新开（见 [README.md](../README.md) 的「Tab 复用模板」）。禁止无脑新开 tab。
2. **环境**：默认测试环境 `10.67.8.183:7777`；正式环境 `51pm.51aes.com:771` 仅在任务明确要求时用，且写操作逐项确认。
3. **登录态**：过期表现=跳登录页；续登/登录见 `regression/scripts/headless-login.js`（launchLoggedIn 封装 SESSION 注入→authorize→落地），失败提示 `npm run login`，不代输凭据。
4. **接口直调**：多数只读取数可跳过 UI 直调接口（页面内 `fetch` 复用会话，或 node 侧带 storageState cookie）。playbook 的「可直调接口」节给的就是这条快路径——**定时/批量任务优先走接口，UI 步骤是兜底与验证**。
5. **视口**：需要出图/看完整右侧栏时，用 headless 脚本（`setViewportSize(1920,1080)` 在集成浏览器会被回弹，不可靠）。

## 单条 playbook 模板

```markdown
# {功能名}【写/只读】

> 一句话：这个流程做完能得到什么。

- **入口**：一级菜单 → 二级页面 → 控件（与 entry_map 对齐，此处只摘要）
- **路由**：`/xxx/yyy`
- **适用/触发**：什么时候会用到（如「每天定时统计上周工时」）

## 操作步骤（真实 UI）

1. 复用/导航到 `/路由`，等 `锚点selector` 出现
2. 点「XX」tab / 按钮
3. 设条件（日期/项目/部门…）—— 注意 {坑}
4. 点「查询」/「提交」
5. 读结果：`selector` 取值 / 预期页面变化

## 可直调接口（免 UI 快路径）

- `GET/POST 接口路径?参数` → 返回什么；边界：{非法值行为}
- 定时任务建议直接用此接口，参数含义见下。

## 读数 / 断言锚点

- 列表主体：`selector`
- 关键字段：`字段名 → 取法`

## 已知坑

- ⚠️ {坑与规避}

---
_来源：V2.2.x 验收轮 ｜ 最后验证：YYYY-MM-DD_
```
