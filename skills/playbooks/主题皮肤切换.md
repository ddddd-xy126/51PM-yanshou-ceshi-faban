# 主题皮肤切换【写】

> 一句话：从顶部主题图标切换全站皮肤（18 个主题），服务端保存跨会话同步。典型 AI 复用场景：「换主题皮肤、验证配色跟随主题变量」。
>
> ⚠️ **写操作**：切换即服务端保存（跨会话生效），测完记得切回原主题。

## 参数

| 参数        | 示例               | 说明                              |
| ----------- | ------------------ | --------------------------------- |
| `env`       | `test` / `prod`    | 测试（默认）/ 正式                |
| `pm_theme`  | `light` / `christmas` | 目标主题枚举（见下 18 项）        |

## 入口

- **入口**：顶部导航主题图标（`.ts-trigger`，emoji 随当前皮肤变）→ 主题面板 popover
- **等待锚点**：`wait_for_selector('.ts-trigger', 15000)`
- **进站**：⚠️ 直接 goto `/main` 只渲染空壳（仅水印），用 `/project/project_list` 作载体

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」。

1. 点顶部主题图标 `.ts-trigger` → 弹主题面板 popover
2. 选主题（18 个：10 基础 + 8 节日 + 2 特效开关）
3. 切换后品牌色/按钮/表头/激活态跟随主题变量；服务端保存，刷新/重进保持

### pm_theme 枚举（18）

`light / spring / midsummer / autumn / winter / arctic / lavender / pink / apple / sunset / spring-festival / lantern-festival / labor-day / doraemon / dragon-boat / mid-autumn / national-day / christmas`

## 步骤

### 1.（UI 复现）打开面板选主题

```js
// 在 /project/project_list 载体页操作（/main 是空壳）
document.querySelector(".ts-trigger")?.click();   // 弹主题面板 popover
// 选目标主题（节日主题动画会令 locator 报「不稳定」，用 force 或坐标点）
Array.from(document.querySelectorAll(".ts-panel .theme-item, .ts-panel [class*=theme]"))
  .find(el => el.offsetParent && el.dataset.theme === "light")?.click();
```

## 🚨 防手滑（写操作）

- 切换即服务端保存、跨会话生效 → 记录原主题，测完切回
- 节日主题动画（圣诞飘雪等）会令 `locator.click` 报「元素不稳定」，用 `force:true` 或坐标点击

## 读数 / 断言锚点

- 主题触发图标：`.ts-trigger`（emoji 随皮肤变）
- 生效断言：品牌色/按钮/表头/激活态 CSS 变量随主题变；刷新后保持
