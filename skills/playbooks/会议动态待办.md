# 会议动态待办（负责人 + 状态）【写】

> 一句话：在项目动态里添加会议动态，其待办事项每条可设负责人与状态。典型 AI 复用场景：「会议纪要跟进待办，指定负责人与进度」。
>
> ⚠️ **写操作**：保存前把每条待办的「内容 + 负责人 + 状态」给用户 AskUserQuestion 确认。

## 参数

| 参数         | 示例                          | 说明                                   |
| ------------ | ----------------------------- | -------------------------------------- |
| `env`        | `test` / `prod`               | 测试（默认）/ 正式；写操作没说就 ask   |
| `project_id` | `6690`                        | 目标项目 id                            |
| `todos`      | `[{内容,负责人,状态}]`        | 待办条目：状态 待处理/进行中/已完成    |

## 入口

- **入口**：项目 → 项目详情 → 项目动态 → 添加动态（会议动态）→ 待办事项
- **等待锚点**：`wait_for_selector('.el-dialog', 15000)`
- **进站**：先按 [../README.md](../README.md) Tab 复用，禁止首选 new_tab

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」。

1. 进项目动态页 → 添加动态（会议动态）
2. 待办事项区添加条目
3. 每条待办设：状态（待处理/进行中/已完成）+ 相关人员
   - ⚠️ 相关人员 el-select 要点 `.el-select__input.is-mini`（tags 拦截坑，同组群弹窗）
4. 保存 → 待办状态与人员正确回读

## 步骤

### 1.（UI 复现）添加会议动态 + 待办

```js
// 相关人员多选：⚠️ 必须点 .el-select__input.is-mini，外层 input 被 tags 拦
const sel = document.querySelector(".el-dialog .el-select__input.is-mini");
sel && sel.focus();   // 再输入/选人；直接点外层 input 无效
```

### 2. 落库校验

```js
// 保存后用接口回读断言（type=1 会议动态，含待办负责人+状态）
// GET /manage_api/project_moment/get_list?project_id=N&type=1 → j.data
```

## 🚨 防手滑（写操作强制）

- 保存前把每条待办「内容 + 负责人 + 状态」给用户 AskUserQuestion 确认
- 相关人员必须点 `.el-select__input.is-mini` 选人，别误点其它下拉
- 一次只保存一条动态，保存后回读校验再建下一条

## 可直调接口

- `project_moment/get_list?project_id=N&type=1`（type=1 会议动态）
  - 返回取值：`j.data`（待办含负责人 + 状态字段，可作落库断言）

## 读数 / 断言锚点

- 待办状态枚举：待处理 / 进行中 / 已完成
- 相关人员选择器：`.el-select__input.is-mini`

## 已知坑

- ⚠️ 相关人员 el-select 需点 `.el-select__input.is-mini`（步骤 1）
