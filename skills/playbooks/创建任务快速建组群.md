# 创建任务时快速建组群并导入成员【写】

> 一句话：在创建任务选指派人的「从组群导入」里就地新建组群、加人、保存，随即导入使用，组群顺手沉淀下次复用。典型 AI 复用场景：「批量指派任务时顺手沉淀常用人员组群」。
>
> ⚠️ **写操作**：加人是「加入 + 保存成员」两步，落库前把组名 + 成员列给用户 AskUserQuestion 确认。

## 参数

| 参数        | 示例            | 说明                              |
| ----------- | --------------- | --------------------------------- |
| `env`       | `test` / `prod` | 测试（默认）/ 正式                |
| `group_name`| `前端常用组`    | 新建组群名                        |
| `members`   | `[张三,李四]`   | 组群成员                          |

## 入口

- **入口**：创建多人通用任务 / 需求拆解向导 →「从组群导入」popover →「立即管理」链接 → 就地弹「组群配置」弹窗
- **等待锚点**：`wait_for_selector('.el-dialog', 15000)`
- **进站**：先按 [../README.md](../README.md) Tab 复用，禁止首选 new_tab

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」。

1. 进入创建任务向导（多人通用任务 / 需求拆解），到指派人环节点「从组群导入」popover
2. popover 内点「立即管理」→ **就地弹出「组群配置」弹窗**（不离开创建流程）
3. 「新建分组」→ 填组名 → 创建成功（自动分配 ID 并选中）
4. 成员下拉真实点击选人 →「加入」→「保存成员」→ toast「保存成功」
   - ⚠️ el-select 多选外层 input 会被 tags 里 mini input 拦截，要点 `.el-select__input.is-mini` / `.is-small` 才有效
5. 关闭组群配置回到创建流程 → 再点「从组群导入」→ 组群下拉出现新组群 → 选中「添加」→ toast「已添加组群『xxx』成员 N 人」，指派人自动填入

## 步骤

### 1.（UI 复现）就地建组群 + 两步加人

```js
// 「从组群导入」popover →「立即管理」→ 组群配置弹窗
Array.from(document.querySelectorAll("a, span, button"))
  .find(el => el.offsetParent && el.textContent.trim() === "立即管理")?.click();
// 成员多选：⚠️ 必须点 mini input，外层 input 被 tags 拦
document.querySelector(".el-dialog .el-select__input.is-mini, .el-dialog .el-select__input.is-small")?.focus();
// 选人后两步：「加入」（进待保存）→「保存成员」（落库）
["加入","保存成员"].forEach(t => Array.from(document.querySelectorAll("button"))
  .find(el => el.offsetParent && el.textContent.trim() === t)?.click());
```

## 🚨 防手滑（写操作强制）

- 加人是**两步**：「加入」（加进待保存）+「保存成员」（落库），漏点第二步不生效
- el-select 多选必须点 `.el-select__input.is-mini` / `.is-small` 选人，别点外层 input
- 保存前把组名 + 成员列给用户 AskUserQuestion 确认

## 读数 / 断言锚点

- 成员选择器：`.el-select__input.is-mini` / `.is-small`
- 落库成功判定：toast「保存成功」；回创建流程组群下拉出现新组群

## 已知坑

- ⚠️ 加人是**两步**：「加入」（加进待保存）+「保存成员」（落库），都要点
