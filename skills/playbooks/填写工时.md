# 填写工时【写】

> 一句话：给某条任务录入一天的工时（花费 + 当日完成工作，可贴图），提交后生成一条「未确认」工时记录。典型 AI 复用场景：「帮我把今天的工时填了、批量给一批任务录工时」。
>
> ⚠️ **写操作**：提交前把「任务 + 工时 + 描述」列给用户 AskUserQuestion 确认，逐条填逐条提交，禁止 agent 自行连点。

## 参数

| 参数        | 示例                        | 说明                                                             |
| ----------- | --------------------------- | --------------------------------------------------------------- |
| `env`       | `test` / `prod`             | 测试 `10.67.8.183:7777`（默认）/ 正式；写操作没说就 ask          |
| `task_kind` | `项目任务` / `非项目任务`   | 两 tab 数据不互通，切 tab 后须重点「查询」                       |
| `task_name` | `平台研发-前端开发`         | 目标任务名（按名定位行，勿用缓存坐标）                           |
| `hours`     | `8`                         | 花费工时                                                        |
| `work_desc` | `完成登录模块联调`          | 当日完成工作（必填）                                            |

## 入口

- **入口**：任务 → 任务列表行 → 操作列 `button.workHour`（icon `el-icon-time`）→ 工时记录列表弹窗 →「填写工时」→ `.el-dialog.workhour-dialog` 表单
- **路由**：项目任务 `/task_panel/project_task`；非项目任务(NPT) `/task_panel/project_not_task`（两 tab 数据不互通）
- **等待锚点**：`wait_for_selector('.el-table', 15000)`
- **进站**：先按 [../README.md](../README.md) Tab 复用，禁止首选 new_tab

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」。

1. 复用/导航到 `/task_panel/project_task`（NPT 走 `/task_panel/project_not_task`），等 `.el-table` 出现
2. 用日期/项目/指派人筛选出目标任务 —— ⚠️ 日期筛选按任务**起止区间相交**匹配，单日 `start=end` 会漏掉跨天任务段；用整周范围查出来再自己挑那条
3. 定位目标行操作列的 `button.workHour`（第 3 个图标，class `workHour`）。⚠️ 固定右列在 `.el-table__fixed-right .el-table__body tbody`，别在主 tbody 找；CDP `locator.click` 可能无效，兜底用原生 `btn.click()`
4. 弹出「工时记录列表」→ 点「填写工时」→ 打开 `.el-dialog.workhour-dialog`
5. 填表单：
   - **花费**：`.el-input-number input`（不要用 `find(value==='8')`，不稳）
   - **当日完成工作**：第一个 `textarea`，**必填**；填值走 native setter + 派发 `input`/`change` 事件
   - （可选）贴图：上传区组件 `UploadImages.vue`，V2.2.5 起支持 Ctrl+V 粘贴
   - **多任务选项任务**：每个子项有独立 `.option-hour-input` 工时框 + 描述 `textarea`；**仅工时 > 0 的子项必填描述**
6. 提交 → 记录状态变「未确认」（确认是另一条流程，不在此）

## 步骤

### 1. 切 tab、按整周筛选、点查询

```js
// 切 tab（两 tab 数据不互通，切后必须重新查询）
Array.from(document.querySelectorAll(".el-tabs__item, span, div"))
  .find(el => el.offsetParent && el.textContent.trim() === task_kind)?.click();
// ⚠️ 日期用整周区间：后端按任务起止「区间相交」匹配，单日会漏跨天任务段
const dateInputs = Array.from(document.querySelectorAll("input.el-input__inner"))
  .filter(el => el.placeholder === "选择日期" && el.offsetParent);
function findPicker(el){ let n=el; while(n){ let v=n.__vue__;
  while(v){ if(v.$options?._componentTag==="el-date-picker") return v; v=v.$parent; } n=n.parentElement; } }
findPicker(dateInputs[0]).$emit("input", new Date("2026-07-13"));
findPicker(dateInputs[1]).$emit("input", new Date("2026-07-19"));
Array.from(document.querySelectorAll("button")).find(b => b.textContent.trim()==="查询" && b.offsetParent)?.click();
```

### 2. 按任务名定位行，点工时按钮（⚠️ 原生 click）

```js
const dedupe = s => { const h=s.length/2; return s.length>1 && s.slice(0,h)===s.slice(h) ? s.slice(0,h):s; };
const body = document.querySelector(".el-table .el-table__body-wrapper .el-table__body tbody");
const rows = Array.from(body.querySelectorAll("tr")).filter(r => r.offsetParent);
const i = rows.findIndex(r => dedupe(r.querySelector("td:first-child .cell")?.textContent.trim()||"") === task_name);
// 操作列在固定右列（别用主 tbody 最后一列）
const fixed = document.querySelector(".el-table__fixed-right .el-table__body tbody");
const btn = fixed.querySelectorAll("tr")[i].querySelector("button.workHour");
btn.click();  // ⚠️ CDP/合成 click 可能无效，用原生 btn.click()
```

### 3. 开「填写工时」表单并填写

```js
Array.from(document.querySelectorAll(".el-dialog button, .el-dialog a"))
  .find(b => b.offsetParent && b.textContent.trim() === "填写工时")?.click();
const dlg = document.querySelector(".el-dialog.workhour-dialog");
const set = (el,v)=>{ const d=Object.getOwnPropertyDescriptor(el.__proto__,'value').set; d.call(el,v);
  el.dispatchEvent(new Event('input',{bubbles:true})); el.dispatchEvent(new Event('change',{bubbles:true})); };
set(dlg.querySelector(".el-input-number input"), String(hours));   // 花费工时
set(dlg.querySelector("textarea"), work_desc);                     // 当日完成工作（必填）
// 贴图（可选）：聚焦上传区后 Ctrl+V（UploadImages.vue）
```

## 🚨 防手滑（写操作强制）

- **逐条填、逐条提交**：每提交 1 条后 wait≥1.5s → 重新点「查询」刷新 → 按**任务名**重新定位下一条；**绝不**缓存多条行坐标批量点（行会重排）
- 操作列只碰 `button.workHour`；**禁止点**编辑（`el-icon-edit-outline`）/ 完工（`button.finish`）/ 删除（`el-icon-delete`）
- 提交前把「任务名 + 工时 + 描述」列给用户 AskUserQuestion 确认；二次确认弹窗交用户点

## 读数 / 断言锚点

- 本页 vm 挂在 `div.main`（`document.querySelector('div.main').__vue__`），**不是** `form.el-form`
- 提交成功判定：工时记录列表出现新行且状态列=「未确认」

## 已知坑

- ⚠️ 排期表任务详情弹窗 `.task-detail-dialog` 是**只读**、没有填工时入口，别在那找
- ⚠️ 日期筛选按任务**区间相交**匹配，单日 start=end 会漏跨天任务段（用整周范围再自己挑）
- ⚠️ 固定右列双份渲染 / `button.workHour` `locator.click` 失效 → 用原生 `btn.click()`
- ⚠️ 花费工时取 `.el-input-number input`（`find value==='8'` 不稳）；当日完成工作=第一个 textarea 必填
- ⚠️ 多任务选项：工时>0 的子项不填描述会被拦
