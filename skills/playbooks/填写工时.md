# 填写工时【写】

> 一句话：给某条任务录入一天的工时（花费 + 当日完成工作，可贴图），提交后生成一条「未确认」工时记录。

- **入口**：任务 → 任务列表行 → 操作列 `button.workHour`（icon `el-icon-time`）→ 工时记录列表弹窗 →「填写工时」→ `.el-dialog.workhour-dialog` 表单
- **路由**：项目任务 `/task_panel/project_task`；非项目任务(NPT) `/task_panel/project_not_task`（两 tab 数据不互通，切 tab 后要重点「查询」）
- **适用/触发**：日常/定时给指定任务补工时；批量为一批任务录工时

## 操作步骤（真实 UI）

1. 复用/导航到 `/task_panel/project_task`（NPT 走 `/task_panel/project_not_task`），等 `.el-table` 出现
2. 用日期/项目/指派人筛选出目标任务 —— ⚠️ 日期筛选按任务**起止区间相交**匹配，单日 `start=end` 会漏掉跨天任务段；用整周范围查出来再自己挑那条
3. 定位目标行操作列的 `button.workHour`（第 3 个图标，class `workHour`）。⚠️ 固定右列在 `.el-table__fixed-right .el-table__body tbody`，别在主 tbody 找；CDP `locator.click` 可能无效，兜底用原生 `btn.click()`
4. 弹出「工时记录列表」→ 点「填写工时」→ 打开 `.el-dialog.workhour-dialog`
5. 填表单：
   - **花费**：`.el-input-number input`（不要用 `find(value==='8')`，不稳）
   - **当日完成工作**：第一个 `textarea`，**必填**；填值走 native setter + 派发 `input`/`change` 事件
   - （可选）贴图：上传区组件 `UploadImages.vue`，V2.2.5 起支持 Ctrl+V 粘贴（自动化可合成 `ClipboardEvent`+`DataTransfer`）
   - **多任务选项任务**：每个子项有独立 `.option-hour-input` 工时框 + 描述 `textarea`；**仅工时 > 0 的子项必填描述**
6. 提交 → 记录状态变「未确认」（确认是另一条流程，不在此）

## 读数 / 断言锚点

- 本页 vm 挂在 `div.main`（`document.querySelector('div.main').__vue__`），**不是** `form.el-form`
- 提交成功判定：工时记录列表出现新行且状态列=「未确认」

## 已知坑

- ⚠️ 排期表任务详情弹窗 `.task-detail-dialog` 是**只读**、没有填工时入口，别在那找
- ⚠️ 单日筛选漏跨天任务段（见步骤 2）
- ⚠️ 固定右列双份渲染 / `locator.click` 失效（见步骤 3）
- ⚠️ 多任务选项：工时>0 的子项不填描述会被拦
