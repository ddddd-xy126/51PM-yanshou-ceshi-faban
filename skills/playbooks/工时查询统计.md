# 工时查询统计 / 每日工作概览【只读】

> 一句话：按时间范围/部门查工时统计，并可导出「每日工作概览」Excel。典型 AI 复用场景：「每天定时拉工时统计、导出每日工作概览发周报」。

## 参数

| 参数         | 示例            | 说明                                             |
| ------------ | --------------- | ------------------------------------------------ |
| `env`        | `test` / `prod` | 测试 `10.67.8.183:7777`（默认）/ 正式；只读默认复用不问 |
| `start_date` | `2026-07-01`    | 统计起始日                                       |
| `end_date`   | `2026-07-31`    | 统计结束日                                       |
| `dept_id[]`  | `12` / 空       | 部门 id 数组，空=全部（产能相关接口）            |

## 入口

- **路由**：⚠️ 精确路由未实测确认（entry_map 无登记）→ 走顶部菜单**统计 → 工时查询统计**进入，别硬拼 URL；下轮验收本功能时把真实路由回填 entry_map 与此处
- **导航**：统计 → 工时查询统计 →「每日工作概览」tab → 右上「导出」
- **相关**：统计 → 产能数据看板（`/statistic/capacity_analysis`）是另一视角的产能统计
- **等待锚点**：`wait_for_selector('.el-table', 15000)`
- **进站**：先按 [../README.md](../README.md) Tab 复用，禁止首选 new_tab

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」，只读定时任务优先走「可直调接口」。

1. 复用/导航到统计模块的「工时查询统计」页，等表格 `.el-table` 出现
2. 切到「每日工作概览」tab
3. 设时间范围（日期区间）/ 部门等筛选条件
   - ⚠️ 日期筛选 `fill` 常不触发查询，走 vm `form` 赋值后调 `search()` / 点「查询」按钮
   - ⚠️ 时间范围过大页面可能出禁用提示 → 先读页面 warning 文案，缩到要求范围再查
4. 点「查询」刷新
5. 需要落文件时点右上「导出」（此页导出**无「包含截图」选项**，可直接下载）

## 步骤

### 1.（推荐·定时任务）接口直调导出每日工作概览

```js
const { request } = require('@playwright/test');
const fs = require('fs');
const state = JSON.parse(fs.readFileSync('auth/state.json', 'utf8'));
const o = state.origins.find(x => x.origin.includes('10.67.8.183'));
const token = o.localStorage.find(l => l.name === 'oauthToken').value;
(async () => {
  const ctx = await request.newContext({
    baseURL: process.env.PM_API_BASE || 'http://localhost:8888',
    extraHTTPHeaders: { Authorization: token, token },
  });
  // 导出每日工作概览（2026-07-17 实测可直接下载，无截图选项）
  const r = await ctx.get('/manage_api/export_daily_estimate');   // 具体 query 下轮验收补
  fs.writeFileSync('每日工作概览.xlsx', await r.body());
  await ctx.dispose();
})();
```

### 2.（UI 复现）切 tab、走 vm 设日期、查询、导出

## 可直调接口（免 UI 快路径 —— 定时任务优先走这条）

- **导出每日工作概览**：`export_daily_estimate`（2026-07-17 实测可直接下载，无截图选项；返回体即 xlsx 二进制）
- **产能数据（相关看板）**：`GET data_export/get_employee_project_list?dept_id[]=&start_date=&end_date=&filter_type=all`
  - 返回取值：`j.data`；边界：非法日期静默兜底不报错；不存在部门返空数组（断言只验不 5xx）
- ⚠️ 「工时查询统计」列表本身的取数接口路径尚未在验收轮确认 → **下次验收本功能时补填此处**（示范：playbook 随验收增量补全）
