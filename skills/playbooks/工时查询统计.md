# 工时查询统计 / 每日工作概览【只读】

> 一句话：按时间范围/部门查工时统计，并可导出「每日工作概览」Excel。是「让 AI 每天定时拉工时统计」这类任务的标准入口。

- **入口**：统计 → 工时查询统计 → 「每日工作概览」→ 右上「导出」
- **相关**：统计 → 产能数据看板（`/statistic/capacity_analysis`）是另一视角的产能统计
- **适用/触发**：定时/按需拉取工时统计；导出每日工作概览发周报

## 操作步骤（真实 UI）

1. 复用/导航到统计模块的「工时查询统计」页，等表格 `.el-table` 出现
2. 切到「每日工作概览」tab
3. 设时间范围（日期区间）/ 部门等筛选条件
   - ⚠️ 日期筛选 `fill` 常不触发查询，走 vm `form` 赋值后调 `search()` / 点「查询」按钮（同 QA 递交、超时递交筛选的已知坑）
   - ⚠️ 时间范围过大页面可能出禁用提示 → 先读页面 warning 文案，缩到要求范围再查
4. 点「查询」刷新
5. 需要落文件时点右上「导出」（此页导出**无「包含截图」选项**，可直接下载）

## 可直调接口（免 UI 快路径 —— 定时任务优先走这条）

- **导出每日工作概览**：`export_daily_estimate`（2026-07-17 实测可直接下载，无截图选项）
- **产能数据（相关看板）**：`GET data_export/get_employee_project_list?dept_id[]=&start_date=&end_date=&filter_type=all`
  - 边界：非法日期静默兜底不报错；不存在部门返空数组（断言只验不 5xx）
- ⚠️ 「工时查询统计」列表本身的取数接口路径尚未在验收轮确认 → **下次验收本功能时补填此处**（示范：playbook 随验收增量补全）

## 读数 / 断言锚点

- 列表主体取数：`.el-table__body-wrapper .el-table__body tbody`（避开固定列双份渲染）
- 单元格文字去重：`s.length>1 && s.slice(0,s.length/2)===s.slice(s.length/2) ? s.slice(0,s.length/2) : s`

## 已知坑

- ⚠️ 日期 `fill` 不触发查询，走 vm form + search（见步骤 3）
- ⚠️ 集成浏览器页内 `fetch` 到 `localhost:8888` 被 CORS preflight 拦，接口直调走 node 侧（带 storageState cookie）
- ⚠️ 「统计 → 工时数据总览」某 tab 是前端 Mock 演示页，不是真实数据，别当统计源
