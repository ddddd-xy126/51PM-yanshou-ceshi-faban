# 我的任务日历（点任务弹花费 / 查看任务描述）【写】

> 一句话：在「我的任务」日历里点任务项直接弹出工时花费填写，点日期格看当日任务卡（含任务描述备注）。典型 AI 复用场景：「按日历补当日工时、快速查某天有哪些任务」。
>
> ⚠️ **写操作**（内含填工时）：提交前把任务 + 工时列给用户 AskUserQuestion 确认，禁止自行连点提交。

## 参数

| 参数        | 示例                | 说明                                     |
| ----------- | ------------------- | ---------------------------------------- |
| `env`       | `test` / `prod`     | 测试（默认）/ 正式；写操作没说就 ask     |
| `date`      | `2026-07-17`        | 目标日期（定位日历格）                   |
| `task_name` | `B/S二开-AI-引擎功能` | 目标任务条名                             |
| `hours`     | `8`                 | 花费工时（填工时时）                     |

## 入口

- **入口**：我的地盘 → 左侧栏「我的任务」→ 日历
- **路由**：`/my_board/main/task`（⚠️ 必须点左侧菜单进入，直接 URL 被重定向回 main）
- **等待锚点**：`wait_for_selector('.tc-cell', 15000)`
- **进站**：先按 [../README.md](../README.md) Tab 复用，禁止首选 new_tab

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」。

1. ⚠️ **必须点左侧菜单进入**，直接 `location.href` / `router.push` 会被重定向回 main
2. 日历格 class `tc-cell`（今天带 `is-today`）
3. **点任务条**（如「B/S二开-AI-引擎功能 | 8H」）→ V2.2.5 起**直接弹工时/花费填写弹窗**（含花费记录列表 +「填写工时」按钮），填写细节见 [填写工时.md](填写工时.md)
4. **点日期格** → 右侧 `tc-panel__body` 侧栏展示当日任务详情（含「快速创建任务 / 工时」入口）
5. 任务卡片「**备注**」字段 = 任务描述原文（V2.2.6 新增；DTA 任务显示全名）

## 步骤

### 1.（UI 复现）进日历、点任务条弹工时

```js
// ⚠️ 必须点左侧菜单，不能直接改 URL
Array.from(document.querySelectorAll(".el-menu-item, .side-menu a, span"))
  .find(el => el.offsetParent && el.textContent.trim() === "我的任务")?.click();
// 点任务条 → 直接弹工时/花费填写弹窗
Array.from(document.querySelectorAll(".tc-cell .task-item, .tc-cell [class*=task]"))
  .find(el => el.offsetParent && el.textContent.includes(task_name))?.click();
// 点日期格看当日任务卡（含「备注」=任务描述）
document.querySelector(".tc-cell.is-today")?.click();   // 右侧 .tc-panel__body 展开
```

### 2. 填工时（写）

打开弹窗后填表 → 见 [填写工时.md](填写工时.md)（花费=`.el-input-number input`、当日完成工作=第一个 textarea 必填）。

## 🚨 防手滑（写操作强制）

- 提交前把「任务名 + 工时 + 描述」列给用户 AskUserQuestion 确认，通过再点提交
- 逐条填、逐条提交，不缓存日历项坐标批量点（日历刷新会重排）
- 二次确认弹窗交用户点，agent 不代点
