# 批量创建任务（通用批量表单向导）【写】

> 一句话：用两步向导 + 侧栏条目管理一次创建多条任务（独立任务=每条字段各异；多人通用=编辑一次对全部指派人生效）。典型 AI 复用场景：「给一个需求批量建任务、多人会议/培训任务批量指派」。
>
> ⚠️ **写操作**：提交前把「共 N 条 + 每条字段」给用户 AskUserQuestion 确认，禁止自行提交。

## 参数

| 参数        | 示例                       | 说明                                         |
| ----------- | -------------------------- | -------------------------------------------- |
| `env`       | `test` / `prod`            | 测试（默认）/ 正式；写操作没说就 ask         |
| `demandId`  | `12345`                    | ⚠️ **必带**，无需求上下文提交静默失败         |
| `kind`      | `独立任务` / `多人通用任务`| 独立=每条字段各异；通用=编辑一次对全部生效   |
| `items`     | `[{任务名,工时,指派人}]`   | 批量条目（空白条目自动丢弃）                 |

## 入口

- **入口**：非项目 → 需求 → 点需求名进任务页 → 左上「创建任务」(悬停) → 独立任务 / 多人通用任务
- **路由**：`/not_project/not_project_task?demandId=N`（实测；⚠️ 必带 demandId）
- **等待锚点**：`wait_for_selector('.el-table', 15000)`
- **进站**：先按 [../README.md](../README.md) Tab 复用，禁止首选 new_tab

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」。

1. 进入需求上下文（**必须带 demandId**，否则提交静默失败）：非项目 → 需求列表 → 点需求名进任务页
2. 左上「创建任务」(悬停展开) → 选「独立任务」或「多人通用任务」
3. 通用批量表单向导（BatchCreateDialog）两步：
   - 第 1 步录入条目：侧栏显示「共 N 条 / 有效 N / 全清」，空白条目自动丢弃
   - 独立任务：每条字段可各不相同；多人通用：编辑一次对全部指派人生效
4. 第 2 步确认 → 提交，toast「成功创建 N 条任务」

## 步骤

### 1.（UI 复现）从需求进入 + 开向导

```js
// ⚠️ 必须从需求页进入（URL 带 demandId）；直接建会静默失败
Array.from(document.querySelectorAll(".create-task, button, span"))
  .find(el => el.offsetParent && el.textContent.trim() === "创建任务")
  ?.dispatchEvent(new MouseEvent("mouseenter", { bubbles: true }));   // 悬停展开
Array.from(document.querySelectorAll("li, a, span"))
  .find(el => el.offsetParent && el.textContent.trim() === "独立任务")?.click();
// 侧栏计数「共 N 条 / 有效 N」，空白条目自动丢弃；走完两步确认再提交
```

## 🚨 防手滑（写操作强制）

- **必须从需求进入**（URL 带 demandId），无上下文提交静默失败（后端 code 51 不透出）
- 提交前把「共 N 条 + 每条任务名/工时/指派人」给用户 AskUserQuestion 确认
- 别误点需求行第一个下拉「F自由需求」（那是建子需求，不是建任务）
