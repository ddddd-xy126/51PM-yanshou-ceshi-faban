# 批量添加项目反馈（通用批量表单向导）【写】

> 一句话：用两步向导批量提交项目反馈申请，选一次项目对全部条目生效。典型 AI 复用场景：「一次性提交多条项目反馈申请」。
>
> ⚠️ **写操作**：提交前把「项目 + 各条商机号/模块/内容」给用户 AskUserQuestion 确认。**造数据一律选有 PM 的项目**（见坑）。

## 参数

| 参数        | 示例                          | 说明                                       |
| ----------- | ----------------------------- | ------------------------------------------ |
| `env`       | `test` / `prod`               | 测试（默认）/ 正式；写操作没说就 ask       |
| `project`   | `新耀 #490`                   | ⚠️ 选**有 PM** 项目，否则提交后不可见无人审 |
| `items`     | `[{商机号,模块,内容}]`        | 批量条目（空白条目自动丢弃）               |

## 入口

- **入口**：我的地盘 → 我的反馈 → 添加反馈 → 批量添加反馈（「批量添加项目反馈」弹窗）
- **等待锚点**：`wait_for_selector('.el-dialog', 15000)`
- **进站**：先按 [../README.md](../README.md) Tab 复用，禁止首选 new_tab

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」。

1. 我的地盘 → 我的反馈 → 添加反馈 → 批量添加反馈
2. 两步向导 + 侧栏条目（共 N 条 / 有效 N / 全清），空白条目自动丢弃
3. 选项目一次对全部条目生效；逐条填商机号/模块/内容
4. 提交 → 落库

## 步骤

### 1.（UI 复现）开向导 + 选有 PM 项目 + 逐条填

```js
// 下拉选项双份渲染 → 过滤 offsetHeight>0 再选
Array.from(document.querySelectorAll(".el-select-dropdown__item"))
  .filter(el => el.offsetHeight > 0)
  .find(el => el.textContent.includes("新耀"))?.click();   // ⚠️ 选有 PM 项目
// 侧栏计数「共 N / 有效 N」，逐条填商机号/模块/内容，走完两步再提交
```

## 🚨 防手滑（写操作强制）

- **造数据/回归一律选有 PM 项目**：无 PM 项目提交成功但不可见且无人可审（V2.2.3 定论）
- 提交前把「项目 + 各条商机号/模块/内容」给用户 AskUserQuestion 确认
- 写链路造数据前先核对项目关键角色（PM/TB）是否齐全，异常先怀疑挑的数据

## 可直调接口

- `produce_demand/add_apply_demand`：空商机号/空模块 → code 51

## 读数 / 断言锚点

- 侧栏计数：共 N 条 / 有效 N / 全清
- 提交成功判定：有 PM 项目立即出现在「我提交的反馈申请」
- 下拉选项去重：过滤 `offsetHeight>0`

## 已知坑

- ⚠️ **提交后可见性看项目是否有 PM**（V2.2.3 定论）：有 PM 项目（如新耀 #490）提交后立即出现在「我提交的反馈申请」；**无 PM 项目**（CBD #488/#489）提交成功但不可见且无人可审——**造数据/回归一律选有 PM 项目**
