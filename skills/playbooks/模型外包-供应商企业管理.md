# 模型外包-供应商企业管理【写】

> 一句话：DTA-PM 在「供应商企业管理」维护供应商账号（新增企业+联系人），并进入供应商门户账密登录入口。V2.2.4 新增，属「场景组-DTA-供应商」全流程中 DTA-PM 的供应商账号管理环节。

- **入口**：我的工作台（顶栏下拉）→「供应商企业管理」；供应商门户 = 我的工作台 →「供应商端入口」
- **路由**：`/supplier_manage`；供应商门户 = `/supplier_portal/login`
- **适用/触发**：需要为新供应商开通账号、或核对供应商企业信息时；供应商开发者自己登录门户填写任务/交付反馈
- **权限**：需 DTA-PM 权限；权限不足页面不可见（超管邓欣羽可见）

## 操作步骤（真实 UI）

1. 复用/导航打开「我的工作台」下拉。⚠️该下拉是 `el-dropdown`，直接 JS `.click()` 触发不稳；可靠开法：对 `.el-dropdown` 触发器依次 dispatch `mouseenter` + `mouseover` + `click`：
   ```js
   const trig = document.querySelector('.el-dropdown .el-dropdown-selfdefine, .el-dropdown [aria-haspopup]');
   ['mouseenter', 'mouseover'].forEach((t) => trig.dispatchEvent(new MouseEvent(t, { bubbles: true })));
   trig.click();
   ```
2. 点下拉里「供应商企业管理」→ 落到 `/supplier_manage`，等表格锚点 `.el-table__body tr` 出现。
3. 新增供应商：点「新增供应商」→ 填 公司名称 / 统一社会信用代码 / 联系人（及电话邮箱等）→ 提交。提交成功后列表新增一行。
4. **创建供应商登录账号**（关键，供应商开发者凭此登录门户）：供应商列表行操作第 3 个「用户管理」图标（person）→「创建用户」→ 填 \*用户名 / \*初始密码 / \*姓名（角色固定「普通成员」）→ 确定→「账号创建成功」。
5. 供应商门户入口（供应商开发者自登录）：我的工作台下拉「供应商端入口」→ 触发 `window.open('/supplier_portal/login')`「供应商制作管理系统」，账号+密码+记住+登录（也可直接导航 `/supplier_portal/login`）。

## 捕获门户 URL（自动化验证）

供应商端入口走 `window.open`，集成/自动化环境需先 override 捕获目标 URL：
```js
await page.addInitScript(() => {
  window.__openedUrls = [];
  const orig = window.open;
  window.open = (u, ...rest) => { window.__openedUrls.push(u); return orig?.call(window, u, ...rest); };
});
// 点「供应商端入口」后读取
const url = await page.evaluate(() => window.__openedUrls.at(-1)); // 期望含 /supplier_portal/login
```

## ✅ 全流程真走查（V2.2.4 端到端实测，R3 已大幅收敛）

以自建测试项目 + 自建供应商账号跑通全链路（均在测试环境授权写操作）：

1. **管理员侧**：新建项目 → 申请发包（照片建模/UE5.5/存放地址/推荐供应商/制作内容）→ 审核「通过审核+确认通过」（状态 OA 流程中）→ 立项「手动创建」选供应商企业+合同金额+报价人天+立项/交付时间（状态“制作中”）→ 前往管理 → 创建任务（资产名/任务类型/“负责供应商用户”自动带出已建账号）。
2. **供应商门户侧**：`/supplier_portal/login` 账密登录 → 跳 `/supplier_portal/projects`（右上角显登录用户名）→ 项目列表（按商机号）→ 发包 `/supplier_portal/outsource_manage/N` → 任务列表（资产/类型/开发者/状态/进度）→ 任务详情弹窗（含创建人=管理员，证明两侧实时同步）。
3. **反馈验收闭环**（管理员侧）：发包详情「反馈管理 → 创建反馈」批量向导（关联任务+模块+内容）→“成功创建”（状态未受理）→全部筛选→“处理”→“通过验收”→确认→状态转**已验收**。
4. **结项（确认结项）**：发包行操作无结项按钮；结项是项目级场景组角色工作项，在项目生命周期末端（开工→初报递交→终报递交→整报→结项）。⭐**不要在 0% 半成品项目上强行结项**（会伪造完整项目状态且不可逆）；如需完整验证在一个走完开工→递交→整改的项目上补验。

## ⚠️ 陷阱与覆盖限制

- 门户走**独立 `supplier_api`**，`get_user_info` 存在 CORS 跨域（证明与主站 `manage_api` 是独立后端）；登录页残留旧会话会先报一次 get_user_info CORS 错，不影响登录（登录接口同源代理可用）。
- **R3（已收敛）**：供应商开发者门户内页已用自建账号真登录验证（项目列表/发包/任务/任务详情可见）；仅“确认结项”因本验收项目刻意停中途未强制执行。
- 创建/处理反馈成功后弹窗常自动重开（re-render 显 stale 旧态），关闭按钮已 invisible 会 timeout，忽略即可。
- 相关环节 playbook：发包立项见 [模型外包-自制发包.md](模型外包-自制发包.md)，反馈验收见 [模型外包-反馈验收工作台.md](模型外包-反馈验收工作台.md)。
