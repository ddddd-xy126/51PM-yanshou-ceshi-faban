# 模型外包 — 反馈验收工作台（含批量创建反馈）【写】

> 一句话：在非自制发包详情的反馈管理里批量创建反馈，再进「验收工作台」用快捷键逐条沉浸式验收。典型 AI 复用场景：「批量处理外包反馈的验收（通过/驳回）」。
>
> ⚠️ **写操作**：通过/驳回均落库，逐条验收前把处理结论给用户确认；快捷键会连续落库。

## 参数

| 参数                   | 示例            | 说明                                          |
| ---------------------- | --------------- | --------------------------------------------- |
| `env`                  | `test` / `prod` | 测试（默认）/ 正式；写操作没说就 ask          |
| `outsource_package_id` | `123`           | 发包 id（⚠️ 非法/缺省接口返全库）             |
| `quality_status`       | `2`             | 0未受理/1修改中/2待验收/3已验收               |

## 入口

- **入口**：项目 → 项目详情 → 模型外包 → 点发包名进发包详情 →「反馈管理」tab →「验收工作台」按钮
- **路由**：`/project/outsource_detail?outsourcePackageId=N`（实测）
- **等待锚点**：`wait_for_selector('.el-table, .el-dialog', 15000)`
- **进站**：先按 [../README.md](../README.md) Tab 复用，禁止首选 new_tab

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」。

### A. 批量创建反馈

1. 反馈管理 tab →「创建反馈」→ 批量向导两步（默认 5 条占位）
2. 每条：选关联任务 + 模块（规范/模型/UV/材质）+ 内容 + Ctrl+V 贴图；侧栏显「有效 N」、下一步计数 (N/5)
3. 确认页复核 →「成功创建 N 条反馈」

### B. 验收工作台（逐条沉浸式）

1. 「验收工作台」按钮（仅拉取待验收 `quality_status=2`，limit=999 不分页）
2. 全屏弹层：顶部快捷键提示 + 进度条「已处理 N/M」，左队列 / 中问题反馈 / 右处理结果
3. 快捷键：`A` 通过（落库 status=3）/ `R` 驳回（理由必填，左Ctrl+Enter 提交，status 2→1，理由存 `remark`）/ `S` 跳过（不计进度）/ `↑↓` 切换
4. 已处理条目按钮禁用防重复；「退出工作台」返回列表自动刷新

## 步骤

### 1.（UI 复现）进验收工作台

```js
// ⚠️ 反馈管理 tab 仅非自制发包显示；「验收工作台」按钮 locator.click 被 hit-target 拦
const btn = Array.from(document.querySelectorAll("button, a"))
  .find(el => el.offsetParent && el.textContent.trim() === "验收工作台");
btn?.focus(); btn && btn.dispatchEvent(new KeyboardEvent("keydown", { key: "Enter", bubbles: true }));  // focus+Enter 兜底
// 空队列会 toast「当前没有待验收的反馈」不打开
```

## 🚨 防手滑（写操作强制）

- 快捷键 `A`/`R` **直接落库**：逐条验收前把「反馈 + 通过/驳回结论」给用户确认，别连按
- 驳回理由必填（左Ctrl+Enter 提交），存 `remark`
- 处理说明/处理截图递交仅供应商门户（supplier_api，独立账密），管理员 token 调返 444，别在管理端试

## 可直调接口

- `outsource_feedback/get_feedback_list`（⚠️ `outsource_package_id` 非法/缺省返**全库**）/ `create_feedback` / `update_feedback_status`（⚠️ 对不存在 id 返成功）/ `get_feedback_detail`
- quality_status: 0未受理 / 1修改中 / 2待验收 / 3已验收

## 读数 / 断言锚点

- 进度条「已处理 N/M」；已处理按钮禁用防重复
- 快捷键：A 通过(status=3) / R 驳回(2→1, remark) / S 跳过 / ↑↓ 切换

## 已知坑

- ⚠️ 反馈管理 tab **仅非自制发包显示**（`hideSelfMade`；`isSelfMade = self_made_dept_name && self_made_uid`）

- ⚠️ 关联任务下拉数据来自 `outsource_task/get_task_list?outsource_package_id=N`，发包下无任务则无数据
- ⚠️ 空队列点验收工作台 → toast「当前没有待验收的反馈」不打开
- 处理说明/处理截图递交仅供应商门户（supplier_api，独立账密），管理员 token 调 444
