# 模型外包 — 反馈验收工作台（含批量创建反馈）【写】

> 一句话：在非自制发包详情的反馈管理里批量创建反馈，再进「验收工作台」用快捷键逐条沉浸式验收。

- **入口**：项目 → 项目详情 → 模型外包 → 点发包名进发包详情 → 「反馈管理」tab → 「验收工作台」按钮
- **路由**：`/project/outsource_detail?outsourcePackageId=N`
- **适用/触发**：批量处理外包反馈的验收（通过/驳回）

## 操作步骤（真实 UI）

### A. 批量创建反馈
1. 反馈管理 tab →「创建反馈」→ 批量向导两步（默认 5 条占位）
2. 每条：选关联任务 + 模块（规范/模型/UV/材质）+ 内容 + Ctrl+V 贴图；侧栏显「有效 N」、下一步计数 (N/5)
3. 确认页复核 → 「成功创建 N 条反馈」

### B. 验收工作台（逐条沉浸式）
1. 「验收工作台」按钮（仅拉取待验收 `quality_status=2`，limit=999 不分页）
2. 全屏弹层：顶部快捷键提示 + 进度条「已处理 N/M」，左队列 / 中问题反馈 / 右处理结果
3. 快捷键：`A` 通过（落库 status=3）/ `R` 驳回（理由必填，左Ctrl+Enter 提交，status 2→1，理由存 `remark`）/ `S` 跳过（不计进度）/ `↑↓` 切换
4. 已处理条目按钮禁用防重复；「退出工作台」返回列表自动刷新

## 可直调接口

- `outsource_feedback/get_feedback_list`（⚠️ `outsource_package_id` 非法/缺省返**全库**）/ `create_feedback` / `update_feedback_status`（⚠️ 对不存在 id 返成功）/ `get_feedback_detail`
- quality_status: 0未受理 / 1修改中 / 2待验收 / 3已验收

## 已知坑

- ⚠️ 反馈管理 tab **仅非自制发包显示**（`hideSelfMade`；`isSelfMade = self_made_dept_name && self_made_uid`）
- ⚠️ 集成浏览器中「验收工作台」按钮 `locator.click` 被 hit-target 拦（info-field__label），用 focus+Enter 兜底
- ⚠️ 关联任务下拉数据来自 `outsource_task/get_task_list?outsource_package_id=N`，发包下无任务则无数据
- ⚠️ 空队列点验收工作台 → toast「当前没有待验收的反馈」不打开
