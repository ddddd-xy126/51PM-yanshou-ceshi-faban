# 模型外包-发包挂起/取消【写】

> 一句话：在编辑发包时把发包置为「挂起」或「取消」——取消的发包不再计入外包成本统计，挂起的发包可随时恢复到制作中或取消。V2.2.9 新增。

- **入口**：项目 → 项目详情 → 模型外包 → 发包操作列 fixed-right「编辑」→「编辑发包」弹窗底部
- **路由**：`/project/outsource_project?projectId=N`
- **适用/触发**：某发包暂缓（挂起，成本先不算）或作废（取消，不计成本），或恢复挂起的发包继续制作

## 参数

| 参数 | 示例 | 说明 |
| --- | --- | --- |
| `projectId` | `6696` | ⚠️示例值，运行时按项目名经 `get_project_list` 解析，或经 `get_package_list` 取带发包的项目 |
| `目标发包` | 「外立面…优化模型制作」 | 按发包名定位其操作行「编辑」 |
| 目标状态 | 挂起 / 取消发包 / 恢复只做 | 挂起→制作中可逆；取消发包为作废 |

## 操作步骤（真实 UI）

1. 复用/导航到 `/project/outsource_project?projectId=N`，等 `.el-table` 出现（`h.waitTableSettled`）
2. （可选）用筛选区「状态筛选」下拉筛选——枚举含 待审核/已驳回/OA流程中/制作中/已结项/**挂起**/**已取消**
3. 在目标发包**操作列（`.el-table__fixed-right`）**点「编辑」（⚠️列表内联那个「编辑」只改外包代号，别用；必须点固定右列的「编辑」开完整表单）
4. 「编辑发包」弹窗（发包信息/立项信息 tab）底部按钮：「挂起」「取消发包」「取消」「确定」
5. 点「挂起」→ 弹「状态变更」确认框「确定将发包状态变更为「挂起」吗？挂起后可随时恢复到制作中」→「确定」提交
   - 取消发包同理点「取消发包」→ 确认框 →「确定」
   - 恢复：对已挂起发包再次编辑，底部出现恢复到制作中/取消的操作
6. 成功后列表状态列更新为「挂起」/「已取消」

```js
// 定位固定右列「编辑」（非内联外包代号编辑）
const editBtn = [...document.querySelectorAll('.el-table__fixed-right button')].find(b => b.innerText.trim() === '编辑');
editBtn.click();
// 弹窗底部挂起
const dlg = [...document.querySelectorAll('.el-dialog')].find(d => /编辑发包/.test(d.innerText) && d.offsetParent !== null);
[...dlg.querySelectorAll('button')].find(b => b.innerText.replace(/\s/g,'') === '挂起').click();
// 状态变更确认框「确定」
[...document.querySelectorAll('.el-dialog')].find(d => /状态变更/.test(d.innerText) && d.offsetParent)
  ?.querySelectorAll('button')?.forEach(b => { if (b.innerText.trim() === '确定') b.click(); });
```

## 可直调接口（读，写接口未反推）

- `GET /manage_api/outsource/get_package_list?page=1&limit=10&sj_num=` → 发包列表，`status` 字段承载新状态（挂起/已取消）；`project_id` 定位所属项目
- ⚠️ 挂起/取消的**写**接口未反推（走 UI 状态变更确认框）；下轮如需自动化写，抓「确定」触发的 POST 反推

## 已知坑

- ⚠️ 列表里有两个「编辑」：内联「编辑」只改**外包代号**；完整「编辑发包」表单在**固定右列**的「编辑」，挂起/取消按钮只在完整表单底部
- ⚠️ 「取消发包不计成本 / 成本按实际只做计算」是后端统计行为，前端不直接可见——需到「统计 → 模型数据看板」`outsource/get_data_overview` 交叉核对发包金额
- ⚠️ 写操作后弹窗常自动重开（re-render），忽略即可
