# 模型外包 — 申请发包 / 自制立项 / 打分【写】

> 一句话：在项目内申请模型发包，立项时可选「自制」流程（指派内部制作组不推外部供应商），立项后可打分评语。典型 AI 复用场景：「发起模型外包/自制发包、立项、结项打分」。
>
> ⚠️ **写操作**：申请/立项/打分均落库，提交前把关键字段给用户 AskUserQuestion 确认。

## 参数

| 参数        | 示例            | 说明                                          |
| ----------- | --------------- | --------------------------------------------- |
| `env`       | `test` / `prod` | 测试（默认）/ 正式；写操作没说就 ask          |
| `projectId` | `6690`          | 路由带的项目 id                               |
| `mode`      | `自制` / `外包` | 立项模式：手动创建选制作组(自制)/报价单拆解(外包) |
| `dept/人`   | `DTA资产/李润芝`| 自制流程级联制作部门/人                       |

## 入口

- **入口**：项目 → 项目详情 →「模型外包」→ 申请发包 / 列表操作行
- **路由**：`/project/outsource_project?projectId=N`（实测）
- **等待锚点**：`wait_for_selector('.el-table', 15000)`
- **进站**：先按 [../README.md](../README.md) Tab 复用，禁止首选 new_tab

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」。

1. 「申请发包」表单：名称 + **发包类型（多选 el-select）** + 引擎版本 + 存放地址 + 制作内容 → 提交
2. 列表操作行「立项」→ 立项弹窗双模式：
   - **报价单拆解**（仅外包）
   - **手动创建**（可指派自制/外包）→ 选「选择制作组(自制)」radio → 提示「当前为自制流程，本次立项不会推送给外部供应商」→ 级联选制作部门/人（如 DTA资产/李润芝）→ 确认立项
3. 立项后列表状态「制作中」+「自制」标签 +「前往管理」+「打分」按钮
4. 「打分」→「结项评价」弹窗（无完工任务时为空态）
5. 「前往管理」→ 发包详情 `/project/outsource_detail?outsourcePackageId=N`（任务表含「质量评分」列、列表/甘特双视图）
