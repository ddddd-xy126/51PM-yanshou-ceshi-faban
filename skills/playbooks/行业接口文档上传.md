# 行业接口文档上传【写】

> 一句话：在项目概况右侧信息栏上传「行业接口文档」，与定制接口文档独立互不覆盖。典型 AI 复用场景：「给项目补充行业接口文档」。
>
> ⚠️ **写操作**：上传前确认文件与目标项目，选文件后**必须**点「确认上传」才落库。

## 参数

| 参数         | 示例            | 说明                          |
| ------------ | --------------- | ----------------------------- |
| `env`        | `test` / `prod` | 测试（默认）/ 正式            |
| `project_id` | `6690`          | 目标项目 id                   |
| `file`       | `接口文档.pdf`  | 上传文件路径                  |

## 入口

- **入口**：项目 → 项目列表 → 点项目名进项目概况 → 右侧项目信息栏「上传行业接口文档」
- **等待锚点**：`wait_for_selector('.info-field, .project-info', 15000)`
- **进站**：先按 [../README.md](../README.md) Tab 复用，禁止首选 new_tab

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」。

1. 进项目概况页
2. 右侧信息栏找「上传行业接口文档」（位于『定制接口文档』与旧文档项之间/下方）
3. 点上传 → 选文件后**还要点「确认上传」**（两步）
4. 上传后文档出现在信息栏，与定制接口文档互不覆盖

## 步骤

### 1.（UI 复现）两步上传

```js
// 点「上传行业接口文档」触发 filechooser（必须消费 setFiles，否则挂起跳回 main）
Array.from(document.querySelectorAll("button, a, span"))
  .find(el => el.offsetParent && el.textContent.includes("上传行业接口文档"))?.click();
// —— Playwright 侧：page.on('filechooser', fc => fc.setFiles(file)) ——
// 选完文件后必须再点「确认上传」才落库
Array.from(document.querySelectorAll("button"))
  .find(el => el.offsetParent && el.textContent.trim() === "确认上传")?.click();
```

## 🚨 防手滑（写操作）

- 上传前确认目标项目与文件，避免传错项目
- 选文件后**必须**点「确认上传」，否则不落库
- filechooser 必须消费（`setFiles`），不消费会挂起并跳回 main → 别留悬空 chooser

## 读数 / 断言锚点

- 成功判定：信息栏出现新「行业接口文档」项，且「定制接口文档」项不变（互不覆盖）

- ⚠️ 上传触发的 filechooser 必须消费（setFiles），不消费会挂起导致页面跳回 main
