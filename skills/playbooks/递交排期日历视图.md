# 递交排期日历视图【只读】

> 一句话：以季度为周期、日历为维度查看递交安排（PM/QA 侧「递交」列表日历视图 + TB 侧「我的递交排期」日历），空间化规划与回顾递交。V2.2.9 新增。

- **入口 A（PM/QA·季度）**：递交 → 递交列表 →「切换至日历视图」，路由 `/OPStestList/OPStestList_list`
- **入口 B（TB·个人）**：我的地盘 → 我的递交 →「我的递交排期」tab，路由 `/my_board/main/tb_publish`
- **适用/触发**：规划未来递交安排 / 回顾历史递交 / TB 分析个人递交排期

## 操作步骤（真实 UI）

### A. 递交列表季度日历视图

1. 复用/导航到 `/OPStestList/OPStestList_list`，等 `.el-table` 出现
2. 点「切换至日历视图」按钮 → 进入「递交日历视图」
3. 顶部季度导航：「上一季度 / 2026 年 Q3（7~9 月）/ 下一季度」；右上「本季度 N 项递交」计数
4. 按月日历（周一~周日）格内展示各递交条目；「返回表格视图」切回列表

### B. 我的递交排期日历

1. 复用/导航到 `/my_board/main/main` → 点左栏「我的递交」→ 点「我的递交排期」tab
2. 顶部状态统计条：全部 / 待递交 / 已逾期 / 按时 / 超时 / 延期
3. 日历格内递交条目按颜色分状态：绿=按时、橙 `rgba(230,162,60)`=延期、红 `rgba(245,108,108)`=超时/逾期
4. 「点击递交条查看详情」查看单条递交详情

```js
// A. 切季度日历视图
[...document.querySelectorAll('button')].find(b => /切换至日历视图/.test(b.innerText))?.click();
// 读本季度计数
document.body.innerText.match(/本季度\s*(\d+)\s*项递交/)?.[1];

// B. 我的递交排期日历（先点左栏「我的递交」→ 再点「我的递交排期」tab）
[...document.querySelectorAll('.el-tabs__item')].find(t => /我的递交排期/.test(t.innerText))?.click();
```

## 可直调接口（读）

- 我的递交排期数据源：`GET /manage_api/main_panel/get_publish_panel`
- 递交列表/季度日历取数走 `project_publish/get_list`（同列表；筛选参数见 entry_map「递交列表超时递交筛选」行）

## 已知坑

- ⚠️ 「我的递交排期」日历条目 `span.proj`（组件 `publishList_TB.vue`）在集成浏览器自动化下 `offsetParent` 为 null，`.click()` 不触发详情弹窗；需真实鼠标点或人工确认详情内容
- ⚠️ 递交列表默认只筛当天，切日历视图前不影响；日历视图按季度取全季数据
