# 需求拆解为任务（含项目会议/项目内审会）【写】

> 一句话：在项目需求页把一条需求拆解成多条任务（走 BatchCreateDialog 两步向导），会议类任务在「任务选项面板」选具体会议类型。典型 AI 复用场景：「把需求分解为可指派任务、建项目会议任务」。
>
> ⚠️ **写操作**：提交前把「共 N 条 + 每条任务名/工时」给用户 AskUserQuestion 确认。

## 参数

| 参数        | 示例                   | 说明                                          |
| ----------- | ---------------------- | --------------------------------------------- |
| `env`       | `test` / `prod`        | 测试（默认）/ 正式；写操作没说就 ask          |
| `projectId` | `6690`                 | 路由带的项目 id                               |
| `demand`    | `某需求`               | 目标需求（行操作列 el-icon-menu 进拆解）      |
| `meeting`   | `项目内审会`           | 会议类：项目启动会/融通会/复盘会/项目内审会   |

## 入口

- **入口**：项目 → 项目详情 → 项目需求 → 需求行操作列 `el-icon-menu` 按钮 → 需求拆解弹窗
- **路由**：`/project/demand?projectId=N`（实测）
- **等待锚点**：`wait_for_selector('.el-table', 15000)`
- **进站**：先按 [../README.md](../README.md) Tab 复用，禁止首选 new_tab

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」。

1. 进项目需求页，找到目标需求行操作列 `el-icon-menu` 按钮
   - ⚠️ 固定列双份渲染，第一个 `el-icon-menu` 不可见，过滤 `offsetWidth>0`
2. 点开「需求拆解弹窗」（BatchCreateDialog 两步向导）
3. 逐级展开「任务选项面板」（task-options-panel-popover）：
   - 会议类：「管理/会议/售前/培训」→ 项目管理 → 项目会议 → 选（项目启动会/融通会/复盘会/**项目内审会**）
   - 勾选后任务名自动生成，如「项目会议-项目内审会 | 0.5H」
4. 默认带任务 2/3/4 空占位卡，多余用 `.bcd-tab__del` 删除
5. 走完两步向导 → 提交，toast「成功创建 N 条任务」

## 步骤

### 1.（UI 复现）进拆解 + 选会议类型

```js
// 需求行操作列 el-icon-menu → ⚠️ 固定列双份渲染，过滤 offsetWidth>0
Array.from(document.querySelectorAll("i.el-icon-menu"))
  .filter(el => el.offsetWidth > 0)[0]?.click();
// 任务选项面板逐级展开选会议类型；多余占位卡用 .bcd-tab__del 删
document.querySelectorAll(".bcd-tab__del").forEach(el => {/* 按需删多余空卡 */});
```

## 🚨 防手滑（写操作强制）

- 提交前把「共 N 条 + 每条任务名/工时」给用户 AskUserQuestion 确认
- 拆解任务工时总和别超父需求标准工时（会被 `preSubmit` 拦，且 toast 偶发不弹=点了没反应）
- 别按 Escape：会关掉整个向导且不保留已填内容

## 读数 / 断言锚点

- 需求行入口：`i.el-icon-menu`（过滤 `offsetWidth>0`）
- 删占位卡：`.bcd-tab__del`
- 提交成功：toast「成功创建 N 条任务」

## 已知坑

- ⚠️ 拆解任务工时总和 > 父需求标准工时会被 `preSubmit` 拦（toast 偶发不弹，表现为点了没反应）
- ⚠️ 任务选项面板「搜索任务选项」**只搜当前分组**（岗位专业/管理会议两分组要先切）
- ⚠️ Escape 会关掉整个向导弹窗且不保留已填内容
