# 项目递交模块（项目内递交记录）【只读】

> 一句话：在项目详情内查看该项目全部递交记录与按时率统计，支持排序。典型 AI 复用场景：「查单个项目递交历史与按时率」。

## 参数

| 参数         | 示例            | 说明                                          |
| ------------ | --------------- | --------------------------------------------- |
| `env`        | `test` / `prod` | 测试（默认）/ 正式；只读默认复用不问          |
| `projectId`  | `6690`          | 路由带的项目 id                               |
| `keywords`   | `某项目`        | 接口按项目名精确匹配                          |

## 入口

- **入口**：项目 → 项目详情 → 二级菜单「项目递交」
- **路由**：`/project/project_publish?projectId=N`（实测）
- **等待锚点**：`wait_for_selector('.el-table, .publish-stat', 15000)`
- **进站**：先按 [../README.md](../README.md) Tab 复用，禁止首选 new_tab

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」。

1. 进项目详情，点二级菜单「项目递交」
2. 顶部统计条：全部 / 按时 / 超时 / 延期 / 待递交（⚠️ 纯统计数字，**不可点击筛选**）
3. 「最新/最早」排序可用（首卡日期随之切换）
4. 空项目显示「该项目暂无递交记录」

## 步骤

### 1.（UI 复现）进模块看统计与排序

```js
// 项目详情内点二级菜单「项目递交」
Array.from(document.querySelectorAll(".el-menu-item, .sub-menu a, span"))
  .find(el => el.offsetParent && el.textContent.trim() === "项目递交")?.click();
// 顶部统计条为纯数字不可点；排序用「最新/最早」按钮
Array.from(document.querySelectorAll("button, .sort-btn"))
  .find(el => el.offsetParent && el.textContent.trim() === "最早")?.click();
```

> ⚠️ 接口直调不可靠（见下），本模块以 UI 读数为准。

## 可直调接口

- `project_publish/get_list?project_id=N`
  - ⚠️ GET 直调时 `project_id` **不参与过滤**（任意 id 返全量约 12810 条）——页面内实际取数走另一通道，接口回归断言不要依赖该参数
  - `keywords` 按项目名精确匹配

## 读数 / 断言锚点

- 顶部统计条：全部 / 按时 / 超时 / 延期 / 待递交（纯数字）
- 空态文案：「该项目暂无递交记录」

## 已知坑

- ⚠️ 顶部数字块纯统计不可点筛选
- ⚠️ 接口 `project_id` GET 直调不过滤（见上），读单项目数据必须走 UI
