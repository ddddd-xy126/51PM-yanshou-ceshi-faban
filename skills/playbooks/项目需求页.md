# 项目需求页 — 查看/需求任务列表/自动暂停【只读】

> 一句话：查项目需求列表及状态，点需求名看其任务列表（含离职人员历史信息），了解「超2周自动暂停」口径。典型 AI 复用场景：「盘点需求状态分布、核对暂停中需求」。

## 参数

| 参数         | 示例                       | 说明                                          |
| ------------ | -------------------------- | --------------------------------------------- |
| `env`        | `test` / `prod`            | 测试（默认）/ 正式；只读默认复用不问          |
| `project_id` | `6690`                     | 目标项目 id（空 → code 51）                   |
| `demand_id`  | `47299`                    | 需求 id（查任务列表用）                        |
| `limit`/`page` | `20` / `1`               | 分页                                          |

## 入口

- **入口**：项目 → 项目详情 → 二级菜单「项目需求」
- **路由**：`/project/demand?projectId=N`（实测）
- **等待锚点**：`wait_for_selector('.el-table', 15000)`
- **进站**：先按 [../README.md](../README.md) Tab 复用，禁止首选 new_tab

## 操作步骤（真实 UI）

> 文字版流程，先建立整体认知；自动化执行看下方「步骤」，只读定时任务优先走「可直调接口」。

1. 进项目需求页（⚠️ 二级菜单点击不路由的坑 V2.2.5 起未复现，`location.href` 可兜底）
2. 需求列表「完工状态」列：进行中 / 未开工 / 已完成 / **暂停中**
   - 「需求描述」列首次点「查看详情」常无响应，需点第二次
3. 点需求名（`span.longText.title.link`）→「需求:xxx」任务列表弹窗（指派给/审核人/工时列）
   - ⚠️ 固定列双份渲染，第一个 span 不可见需过滤 `offsetWidth>0`
   - 离职人员历史任务的「指派给」列**仍正常显示姓名**（V2.2.7 修复点，不再空白）

### 「超2周自动暂停」口径（V2.2.8）

- 项目下**进行中的新增/变更/初始需求**，若近 2 周无新增任务且无填工时 → 自动转 `pause`（UI 显「暂停中」）
- 样本：贡井区国资管理平台(#6690) 需求 #47299/#47289
- ⚠️ 定时任务触发周期与「两周内是否填过工时」的排除逻辑无法从前端验证

## 步骤

### 1.（推荐·定时任务）接口直调需求列表，秒出数据

```js
const { request } = require('@playwright/test');
const fs = require('fs');
const state = JSON.parse(fs.readFileSync('auth/state.json', 'utf8'));
const o = state.origins.find(x => x.origin.includes('10.67.8.183'));
const token = o.localStorage.find(l => l.name === 'oauthToken').value;
(async () => {
  const ctx = await request.newContext({
    baseURL: process.env.PM_API_BASE || 'http://localhost:8888',
    extraHTTPHeaders: { Authorization: token, token },
  });
  const r = await ctx.get('/manage_api/demand/get_project_demand_list?project_id=6690&limit=100&page=1');
  const j = await r.json();
  const rows = j.data?.list || j.data || [];
  console.log('暂停中需求', rows.filter(x => x.status === 'pause').length, '/ 总', rows.length);
  await ctx.dispose();
})();
```

### 2.（UI 复现）点需求名看任务列表

```js
// 点需求名 span.longText.title.link → 任务列表弹窗
Array.from(document.querySelectorAll("span.longText.title.link"))
  .filter(el => el.offsetWidth > 0)[0]?.click();   // ⚠️ 固定列双份渲染，过滤 offsetWidth>0
```

## 可直调接口（免 UI 快路径 —— 定时/只读优先）

- `demand/get_project_demand_list?project_id=N&limit=&page=`
  - 返回取值：`j.data`（list/数组）；status 枚举 `doing/done/wait/pause`
- `project_task/get_task_list_by_demand_id?demand_id=N`
  - 返回取值：`assigned_to` 人员 id（离职人员 id 完整保留）；空参 code 51
- 离职人员识别 = `user/get_user_select_list`（在职）与 `get_all=true`（全量）的差集

## 读数 / 断言锚点

- 完工状态列枚举：进行中/未开工/已完成/暂停中（`doing/wait/done/pause`）
- 需求名可点：`span.longText.title.link`（过滤 `offsetWidth>0`）
